## v0.4.0 - 2026-07-04

### Added
- §43.9 goal 실행 프로토콜(Goal Execution Protocol) 신설 — 코드·기능 변경(goal) 작업을 분석·설계(강한 모델) → 구현(Sonnet 등 경량 모델 + ultracode/Workflow 멀티에이전트) → 감사(강한 모델)의 3단계로 구조화
  - 1단계에서 데이터 흐름·구현 지점을 파악하고 계약(변경 파일·시그니처·데이터 형태·시드)으로 고정, 2단계에서 계약과 정확한 앵커·코드로 독립 영역을 병렬 구현, 3단계에서 전체 diff를 계약과 1:1 대조하고 실행 검증 후 완료 보고
  - 기존 §43 모델 하네스(§43.1–§43.8)를 대체하지 않고 그 위에서 실행 구조를 정하며 §3·§16·§27·§40과 연계

## v0.3.0 - 2026-07-04

### Added
- §43 모델 하네스(Model-Tier Harness) 신설 — 최상위 모델에서 하위·경량 모델(Opus, Sonnet 등)로 전환되어도 동일한 작업 품질을 유지하기 위한 절차적 보정 규칙
  - 계획 분해(micro-task + 단계별 완료 기준), 근거 우선(기억·추측 대신 실제 확인), 작은 변경과 즉시 검증
  - 상태 외부화(진행 상황·확인된 사실을 문서에 기록), 자기 검증 루프(§16 연계), 실패 처리(3회 실패 시 접근 변경, 그래도 해결되지 않으면 `BLOCKED` 보고)
  - 모델 사용 전략(계획·리뷰는 강한 모델, 기계적 구현은 경량 모델 + 하네스 엄격 적용), 모델 등급을 품질 저하의 사유로 삼지 않음
